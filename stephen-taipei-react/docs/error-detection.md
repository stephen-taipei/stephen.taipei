# 錯誤頁面檢測功能

## 功能說明

截圖腳本會自動檢測並跳過「不存在」的頁面，避免截取錯誤畫面。

## 檢測規則

腳本會檢查頁面內容是否包含以下關鍵字：

- ✓ `不存在`

**說明**：只檢測「不存在」文字，避免誤判正常頁面的「返回」按鈕。

## 範例

### 會被跳過的頁面
- https://stephen.taipei/open-source/ai-local-tools/620-area-chart
  - 顯示：「工具不存在 返回工具平台」
  - 結果：**自動跳過，不截圖**

### 正常處理的頁面
- https://stephen.taipei/open-source/tailwind-templates/card-001
  - 顯示：正常的 UI 元件
  - 結果：**正常截圖**

## 統計資訊

執行完成後會顯示：
```
總項目: 7793
成功截圖: 6500
工具不存在: 150 (已自動跳過)
載入失敗: 80
已存在跳過: 1063
```

## 測試

可以執行測試腳本驗證檢測功能：
```bash
node test-error-detection.js
```

## 注意事項

- 被跳過的頁面不會產生截圖檔案
- 會計入 `工具不存在` 統計中
- 不會影響後續項目的處理
